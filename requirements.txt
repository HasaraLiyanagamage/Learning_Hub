1. Project Overview
Domain: Education & Learning
Platform: Cross-platform Mobile Application
Framework: Flutter (Dart)
State Management: Provider (or Riverpod)
Local Database: SQLite (sqflite)
Backend Service: Firebase Firestore / REST API
Architecture: Clean Architecture with Feature-Based Folder Structure
The Smart Learning Assistant Platform is a mobile application designed to help students learn effectively through courses, lessons, quizzes, notes, progress tracking, notifications, and an AI-powered chatbot. The application supports partial offline access, full CRUD operations, backend interaction, and data synchronization, fulfilling all requirements of the Mobile Application Development Coursework.

2. Objectives of the Application
* Provide an interactive learning platform for students
* Support offline learning using SQLite
* Synchronize data with a backend service when online
* Implement full CRUD functionality
* Demonstrate proper architecture, state management, and UI/UX best practices
* Include both User (Student) and Administrator modules

9. Offline & Online Data Synchronization
Strategy (Partial Offline Support)
* Online: Fetch data from Firebase ? Cache in SQLite
* Offline: Read/write from SQLite only
* Reconnect: Sync pending records using timestamps
This satisfies the offline + online data flow requirement.
